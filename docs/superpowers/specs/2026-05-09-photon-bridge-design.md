# IPhotonBridge — LobbyManager State Machine Testing

**Date:** 2026-05-09
**Goal:** Enable unit testing of `LobbyManager`'s full state machine (join, invite join, leave, rejoin, recreate, player tracking, version drift) without running the game or BepInEx.

## Problem

`LobbyManager` takes `IMultiplayerController` — an IL2CPP interop interface generated by BepInEx from the game binary. It cannot be implemented or mocked in managed code outside the game runtime. Currently only `IsValidRoomName` (a pure static method) is testable.

## Solution

Introduce `IPhotonBridge` — a managed abstraction over `IMultiplayerController`. All IL2CPP translation is confined to `PhotonControllerBridge`. `LobbyManager` talks only to the managed interface. Tests use `FakePhotonBridge`.

## Component Design

### `PlayerInfo` record

```csharp
public record PlayerInfo(string UserId, string? Version = null);
```

Managed replacement for `NetworkPlayer`. Used in all player-related events and `GetRoomPlayers()`.

### `IPhotonBridge`

```csharp
public interface IPhotonBridge
{
    void JoinRoom(string roomName);
    void CreateRoom(string name);        // room options baked into production bridge
    void LeaveRoom();
    void JoinRandomRoom();
    void AllowKickPlayers(bool allow);
    void KickPlayer(string userId);
    void SetLocalVersion(string version); // replaces PhotonPropertyHelper.SetLocalVersion

    string? GetRoomName();
    IReadOnlyList<PlayerInfo> GetRoomPlayers();
    bool IsMasterClient();

    event Action<PlayerInfo> PlayerEntered;
    event Action<PlayerInfo> PlayerLeft;
    event Action<PlayerInfo> PlayerPropertiesChanged;  // carries new Version on the PlayerInfo
}
```

### `PhotonControllerBridge`

Production implementation. Only file that imports IL2CPP types. Responsibilities:

- Subscribes to `add_OnPlayerEnteredRoomEvent`, `add_OnPlayerLeftRoomEvent`, `add_OnPlayerPropertiesChangedEvent` on the controller.
- Translates each `NetworkPlayer` to `PlayerInfo`, reading `PhotonPropertyHelper.ReadPeerVersion(p)` before firing the managed event.
- Bakes fixed room options into `CreateRoom` (`publishUserId=false`, `isOpen=true`, `isVisible=false`, `maxPlayers=5`, `playerTtl=60000`, `emptyRoomTtl=0`).
- `GetRoomPlayers()` calls `_controller.GetRoomPlayers()`, iterates the IL2CPP array, and returns `List<PlayerInfo>` with versions filled in.
- `SetLocalVersion` delegates to `PhotonPropertyHelper.SetLocalVersion`.

`Plugin.cs` constructs the bridge before calling `Initialize`:

```csharp
var bridge = new PhotonControllerBridge(controller);
LobbyManager.Initialize(bridge);
```

### `LobbyManager` changes

Four mechanical changes — no logic altered:

1. **Constructor** — `private` → `internal`, parameter type `IMultiplayerController` → `IPhotonBridge`, field `_controller` → `_bridge`. Event subscriptions become `_bridge.PlayerEntered += HandlePlayerEntered` etc.
2. **Handler signatures** — `HandlePlayerEntered`, `HandlePlayerLeft`, `HandlePlayerPropertiesUpdate` take `PlayerInfo` instead of `NetworkPlayer`. `PhotonPropertyHelper.ReadPeerVersion` calls removed (version is already on the `PlayerInfo`).
3. **Call sites** — all `_controller.XYZ(...)` renamed to `_bridge.XYZ(...)`. `PhotonPropertyHelper.SetLocalVersion(...)` in `HandleJoinedRoom` becomes `_bridge.SetLocalVersion(...)`. `RefreshRoomPlayers` and `RebuildVersionMap` use `_bridge.GetRoomPlayers()`.
4. **Null-safe logging** — `Plugin.Log.LogX(...)` → `Plugin.Log?.LogX(...)` throughout `LobbyManager`.

### `FakePhotonBridge` (tests only)

Pure state driver — no call recording. Controllable return values and event-firing helpers:

```csharp
public class FakePhotonBridge : IPhotonBridge
{
    public string? CurrentRoomName { get; set; }
    public bool MasterClient { get; set; } = true;
    public List<PlayerInfo> RoomPlayers { get; } = new();

    public event Action<PlayerInfo>? PlayerEntered;
    public event Action<PlayerInfo>? PlayerLeft;
    public event Action<PlayerInfo>? PlayerPropertiesChanged;

    public string? GetRoomName() => CurrentRoomName;
    public IReadOnlyList<PlayerInfo> GetRoomPlayers() => RoomPlayers;
    public bool IsMasterClient() => MasterClient;

    public void JoinRoom(string roomName) { }
    public void CreateRoom(string name) { }
    public void LeaveRoom() { }
    public void JoinRandomRoom() { }
    public void AllowKickPlayers(bool allow) { }
    public void KickPlayer(string userId) { }
    public void SetLocalVersion(string version) { }

    public void FirePlayerEntered(PlayerInfo p) => PlayerEntered?.Invoke(p);
    public void FirePlayerLeft(PlayerInfo p) => PlayerLeft?.Invoke(p);
    public void FirePlayerPropertiesChanged(PlayerInfo p) => PlayerPropertiesChanged?.Invoke(p);
}
```

## Test Patterns

Tests call `internal` handler methods directly (Harmony does not run in tests). Player events are driven via the fake's Fire methods.

```csharp
// Setup (in test class constructor or fixture)
ConfigStore.OverridePath = Path.Combine(Path.GetTempPath(), Path.GetRandomFileName());
var fake = new FakePhotonBridge { CurrentRoomName = "MY-ROOM" };
var manager = new LobbyManager(fake);
// HandleJoinedRoom distinguishes home vs. game rooms by comparing GetRoomName() to
// Config.LobbyRoomName. Set Config.LobbyRoomName after construction to match your test scenario.
manager.Config.LobbyRoomName = "MY-ROOM";

// Invite join — home lobby name restored after join
manager.JoinRoomByInvite("FRIEND-ROOM");
fake.CurrentRoomName = "FRIEND-ROOM";
manager.HandleJoinedRoom();
Assert.Equal("MY-ROOM", manager.Config.LobbyRoomName);

// Invite join with create (host not present) — home lobby name still restored
manager.JoinRoomByInvite("FRIEND-ROOM");
manager.HandleJoinRoomFailed(32758, "Room not found");
fake.CurrentRoomName = "FRIEND-ROOM";
manager.HandleCreatedRoom();
Assert.Equal("MY-ROOM", manager.Config.LobbyRoomName);

// Game flow — post-game prompt shown after game ends
manager.HandleJoinedRoom();           // enter home lobby
fake.CurrentRoomName = "GAME-ROOM";
manager.HandleJoinedRoom();           // enter game room
manager.HandleLeftRoom();             // game ends normally
Assert.True(manager.PendingRejoin);
Assert.False(manager.InGame);

// Mid-game leave — no post-game prompt
manager.HandleJoinedRoom();
fake.CurrentRoomName = "GAME-ROOM";
manager.HandleJoinedRoom();
manager.LeaveToHomeLobby();           // explicit leave mid-game
manager.HandleLeftRoom();
Assert.False(manager.PendingRejoin);

// Version drift — peer with older version
fake.FirePlayerEntered(new PlayerInfo("STEAM123", "1.0.0"));
Assert.True(manager.HasVersionDrift);

// Version update via properties changed
fake.FirePlayerPropertiesChanged(new PlayerInfo("STEAM123", Plugin.PluginVersion));
Assert.False(manager.HasVersionDrift);

// Player tracking
fake.FirePlayerEntered(new PlayerInfo("STEAM456"));
Assert.Contains("STEAM456", manager.RoomSteamIds);
fake.FirePlayerLeft(new PlayerInfo("STEAM456"));
Assert.DoesNotContain("STEAM456", manager.RoomSteamIds);
```

## What This Does Not Cover

- Real Photon network behaviour (room capacity, actual join failures, network latency).
- Steam invite delivery (`SteamJoinPatch`, `SteamInviter`) — still requires a live game session.
- `OverlayPanel` rendering — Unity MonoBehaviour, not testable outside the game.

These remain manual / in-game verification.
