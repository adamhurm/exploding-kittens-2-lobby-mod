**Goal:** Add a 5-second auto-rejoin countdown after a game ends and a "X in party" presence indicator in the minimized overlay tab.

**Architecture:** All changes live in two existing files — `LobbyManager.cs` gets two new properties (`AutoQueueActive`, `AutoQueueEnabled`) and a new event (`AutoQueueCancelled`); `OverlayPanel.cs` gets a countdown coroutine, a countdown label, and a party dot+text pair in the minimized tab. ## Task 1: LobbyManager — Auto-Queue State and Cancellation Event

**Files:**
- Modify: `src/EKLobbyMod/LobbyManager.cs`

### Background

`HandleLeftRoom()` currently always sets `PendingRejoin = true` and fires `RejoinAvailable`.
We need it to also set `AutoQueueActive = true` on the first leave (natural game end), and
detect a *second* `OnLeftRoom` while `AutoQueueActive` is true as an explicit "player chose
to leave" — in that case it fires `AutoQueueCancelled` instead and resets state.

`HandleJoinedRoom()` must clear `AutoQueueActive` so the flag does not persist after a
successful rejoin.

- [ ] **Step 1: Add `AutoQueueActive`, `AutoQueueEnabled`, and `AutoQueueCancelled` to the public surface of `LobbyManager`**

  In `src/EKLobbyMod/LobbyManager.cs`, locate the existing event declarations (lines 27–29):

  ```csharp
  public event System.Action RejoinAvailable;
  public event System.Action RejoinConfirmed;
  public event System.Action PlayerListChanged;
  ```

  Add the new event and two properties immediately after those three lines:

  ```csharp
  public event System.Action AutoQueueCancelled;

  // True while the 5-second countdown is running in the overlay.
  // Reset to false on OnJoinedRoom or when the player explicitly leaves mid-countdown.
  public bool AutoQueueActive { get; private set; }

  // Set to false to skip the auto-queue countdown entirely (reserved for future settings UI).
  public bool AutoQueueEnabled { get; set; } = true;
  ```

- [ ] **Step 2: Replace `HandleLeftRoom` with the two-scenario version**

  The current `HandleLeftRoom` body (lines 124–130) is:

  ```csharp
  internal void HandleLeftRoom()
  {
      PendingRejoin = true;
      _roomSteamIds.Clear();
      PlayerListChanged?.Invoke();
      RejoinAvailable?.Invoke();
      Plugin.Log.LogInfo("Left room — rejoin prompt raised");
  }
  ```

  Replace it entirely with:

  ```csharp
  internal void HandleLeftRoom()
  {
      if (AutoQueueActive)
      {
          // A second OnLeftRoom while the countdown is running means the player clicked
          // the game's native Leave button — treat this as an explicit opt-out.
          AutoQueueActive = false;
          PendingRejoin = false;
          _roomSteamIds.Clear();
          PlayerListChanged?.Invoke();
          AutoQueueCancelled?.Invoke();
          Plugin.Log.LogInfo("Left room during countdown — auto-queue cancelled");
          return;
      }

      PendingRejoin = true;
      if (AutoQueueEnabled)
          AutoQueueActive = true;
      _roomSteamIds.Clear();
      PlayerListChanged?.Invoke();
      RejoinAvailable?.Invoke();
      Plugin.Log.LogInfo("Left room — rejoin prompt raised");
  }
  ```

- [ ] **Step 3: Update `HandleJoinedRoom` to clear `AutoQueueActive`**

  The current `HandleJoinedRoom` body (lines 109–121) is:

## Task 3: OverlayPanel — Countdown Coroutine and ShowRejoinPrompt / HideRejoinPrompt Updates

- [ ] **Step 1: Add the `RunCountdown` coroutine method**

  `// ── uGUI helpers ──` comment block (after `OpenFriendPicker`, around line 381):

  ```csharp
  private System.Collections.IEnumerator RunCountdown()
  {
      for (int i = 5; i >= 1; i--)
      {
          _countdownLabel.gameObject.SetActive(true);
          _countdownLabel.text = $"Rejoining in {i}…";
          yield return new WaitForSeconds(1f);

          if (!_manager.AutoQueueActive)
          {
              // Cancelled externally (AutoQueueCancelled event fired, flag already cleared)
              _countdownLabel.gameObject.SetActive(false);
              yield break;
          }
      }
      _countdownLabel.gameObject.SetActive(false);
      _manager.JoinOrCreateHomeLobby();
  }
  ```

  `…` is the Unicode ellipsis character `…` — matches the spec display text.
